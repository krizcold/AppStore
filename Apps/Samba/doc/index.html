üõ°Ô∏è Enhanced Security Recommendation

For maximum security, especially when accessing from outside your home network, use Tailscale VPN and bind SMB to your Tailscale IP address.

TailscaleSetup Instructions

  1. Install Tailscale on your server and client devices
  2. Find your Tailscale IP: Run tailscale ip -4 on your server
  3. Secure Configuration: Edit the SMB docker-compose.yml to bind only to Tailscale IP

üîß Docker Compose Modification

Change the ports section in your docker-compose.yml from:

ports: - 445:445

To bind only to your Tailscale IP (example):

ports: - 100.64.1.100:445:445
Security Benefit: This prevents SMB access from your local network and only allows connections through Tailscale VPN, providing encrypted access from anywhere.

üåê Accessing via Tailscale

Once configured, access your SMB share using your Tailscale IP:

Windows:

\\100.64.1.100\DATA

macOS:

smb://100.64.1.100/DATA

Linux:

smb://100.64.1.100/DATA
Replace 100.64.1.100 with your actual Tailscale IP address. This provides secure, encrypted access to your files from anywhere in the world.

SMB Protocol: Port 445 (standard SMB port)

Important: SMB operates as a native network protocol and integrates directly with your operating system's file manager. No web browser interface is needed or provided.

üîç Troubleshooting

  • Connection Issues: Verify the server IP address and ensure port 445 is accessible
  • Permission Denied: Use the correct username (admin) and password
  • Network Discovery: On Windows, SMB shares may appear automatically in Network locations
  • Firewall: Ensure your firewall allows SMB traffic on port 445
  • Tailscale Issues: Verify both devices are connected to the same Tailscale network
